// Calcula el promedio de 'cantEst' estudiantes, con 'cantNotas' notas cada uno. // El núcleo son dos ciclos repetir…hasta que, uno dentro del otro.

El corazón del programa son dos ciclos anidados. El de afuera avanza estudiante por estudiante (se repite cantEst veces). El de adentro avanza nota por nota de ese estudiante (se repite cantNotas veces).

Por qué se reinician promedioN y j: están dentro del ciclo externo, así que vuelven a 0 al empezar cada estudiante (líneas 19).

Por qué promedioE no se reinicia: se pone en 0 una sola vez, antes del ciclo (línea 15). Por eso conserva lo acumulado de todos.

Truco del algoritmo: divide entre j y entre i porque, al salir de cada ciclo, j vale cuántas notas hubo y i cuántos estudiantes hubo.

cantEst · cuántos estudiantes hay
cantNotas · cuántas notas tiene cada uno
i · estudiante actual (ciclo externo)
j · nota actual (ciclo interno) — se reinicia
nota · la nota que se acaba de leer
promedioN · de UN estudiante — se reinicia
promedioE · acumula a TODOS — no se reinicia
resp · s/n para repetir el programa

El mismo ciclo interno, en 3 formas

mostrar ▼

El algoritmo se llama UsosDeCiclos y su comentario dice que usa repetir, para y mientras… pero en realidad lo implementó todo con repetir…hasta que. Estas tres versiones del ciclo de notas hacen exactamente lo mismo:

repetir (el original)

j <- 0 repetir j <- j+1 leer nota promedioN <- promedioN+nota hasta que (j >= cantNotas)

Revisa la condición al final: el cuerpo siempre corre al menos una vez.

para

promedioN <- 0 Para j<-1 hasta cantNotas hacer leer nota promedioN <- promedioN+nota FinPara

El contador j sube solo. Ideal cuando sabes cuántas vueltas serán.

mientras

j <- 0 Mientras (j < cantNotas) hacer j <- j+1 leer nota promedioN <- promedioN+nota FinMientras

Revisa la condición al principio: si fuera 0, no entraría nunca.

Correctas: 0 / 0
